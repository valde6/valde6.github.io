SQL SELECT JOIN UNION INTERSECT EXCEPT Subquery GROUP BY Aggregering

Genereret 28. marts 2026

📋 Opsummering

Forelæsning 17 dykker ned i SQL-forespørgsler fra simple single-table queries til komplekse multi-table operationer. Den grundlæggende SELECT-anatomi er SELECT ... FROM ... WHERE ... ORDER BY, hvortil man kan tilføje DISTINCT for at fjerne dubletter, aggregeringsfunktioner som COUNT, MIN, MAX, SUM og AVG, samt GROUP BY for at gruppere resultater.

Multi-table forespørgsler bygger på mængdeteori: relationer ses som mængder, og man kan kombinere dem via JOIN (inner join / equijoin), UNION (foreningsmængde), INTERSECT (fællesmængde) og EXCEPT/MINUS (differensmængde).

Subqueries giver mulighed for at indlejre én forespørgsel inden i en anden — resultatet af en forespørgsel er altid en tabel, der kan bruges som input til en anden forespørgsel.

🔑 Vigtige begreber

Begreb Definition Eksempel
SELECT-FROM-WHERE Den grundlæggende SQL-forespørgsels-anatomi. SELECT projicerer kolonner, FROM angiver tabel, WHERE filtrerer rækker. SELECT ename FROM emp WHERE sal > 2000
DISTINCT Fjerner dubletter fra resultatet. SELECT DISTINCT deptno FROM emp
ORDER BY Sorterer resultatet. ASC = stigende (default), DESC = faldende. ORDER BY deptno ASC, hiredate DESC
GROUP BY Grupperer rækker med samme værdi i en kolonne — bruges med aggregeringsfunktioner. GROUP BY deptno → min/max løn per afdeling
Aggregeringsfunktioner COUNT, MIN, MAX, SUM, AVG — beregner værdier over en gruppe rækker. Ignorerer NULL (undtagen COUNT). SELECT count(*), avg(sal) FROM emp
INNER JOIN Kombinerer rækker fra to tabeller baseret på en betingelse (equijoin). Kun rækker med match i begge tabeller inkluderes. FROM emp INNER JOIN dept ON emp.deptno = dept.deptno
Subquery En forespørgsel indlejret i en anden. Resultatet bruges som tabel eller betingelsesværdi. WHERE deptno = (SELECT deptno FROM dept WHERE dname='SALES')
UNION Foreningsmængde: kombinerer to tabellers rækker og fjerner dubletter. Tabellerne skal være union-kompatible. SELECT ... UNION SELECT ...
INTERSECT Fællesmængde: returnerer kun rækker der optræder i begge tabeller. SELECT ... INTERSECT SELECT ...
EXCEPT Differensmængde: rækker i første tabel men ikke i anden. Asymmetrisk — rækkefølgen betyder noget. SELECT ... EXCEPT SELECT ...

🎯 Til eksamen

  • SELECT-anatomi: SELECT [DISTINCT] kolonner FROM tabel [WHERE betingelse] [ORDER BY kolonner] — skriv det udenad.
  • JOIN vs. subquery: Begge kan give samme resultat. JOIN kombinerer tabeller, subquery indlejrer én forespørgsel i en anden. Kend begge syntakser.
  • Aggregering: COUNT tæller alle rækker inkl. NULL. MIN, MAX, AVG, SUM ignorerer NULL. Brug GROUP BY til at aggregere per gruppe.
  • Mængdeoperationer: UNION fjerner dubletter (UNION ALL beholder dem). INTERSECT = fælles rækker. EXCEPT = asymmetrisk differens. Tabellerne skal være union-kompatible.
  • INNER JOIN-syntaks: FROM emp INNER JOIN dept ON emp.deptno = dept.deptno — kend den.
  • Subquery-logik: Forstå den indefra-ud — den inderste forespørgsel køres først og leverer input til den ydre.
  • BETWEEN, IN, LIKE, IS NULL: Kend de vigtigste sammenligningsoperatorer i WHERE.
📐 SQL-syntaks reference

SELECT-anatomi

SELECT [DISTINCT] kolonne(r) [AS alias]
FROM tabel
[WHERE betingelse]
[GROUP BY kolonne(r)]
[ORDER BY kolonne(r) [ASC | DESC]];

JOIN

-- Implicit join
SELECT ename, dname
FROM emp, dept
WHERE emp.deptno = dept.deptno
  AND job = 'SALESMAN';

-- INNER JOIN (anbefalet syntaks)
SELECT ename, dname
FROM emp INNER JOIN dept
  ON emp.deptno = dept.deptno
WHERE job = 'SALESMAN';

Subquery

-- Simpel subquery
SELECT ename
FROM emp
WHERE deptno = (
  SELECT deptno FROM dept
  WHERE dname = 'SALES'
);

-- Nested subquery (indefra-ud)
SELECT ename, sal
FROM emp
WHERE sal = (
  SELECT max(sal) FROM emp
  WHERE deptno = (
    SELECT deptno FROM dept
    WHERE dname = 'SALES'
  )
);

Aggregering og GROUP BY

SELECT deptno, min(SAL), max(SAL), avg(SAL)
FROM emp
GROUP BY deptno;

SELECT count(*) FROM emp;
SELECT count(DISTINCT JOB) FROM emp;

Mængdeoperationer

-- UNION (fjerner dubletter)
SELECT * FROM emp UNION SELECT * FROM emp2;

-- INTERSECT (fælles rækker)
SELECT * FROM emp INTERSECT SELECT * FROM emp2;

-- EXCEPT (kun i første tabel)
SELECT * FROM emp EXCEPT SELECT * FROM emp2;

📖 Uddybning

WHERE-betingelser og operatorer

Sammenligningsoperatorer: =, != eller <>, <, >, <=, >=

IN: WHERE deptno IN (10, 20) — svarer til OR med flere værdier.

BETWEEN: WHERE sal BETWEEN 1500 AND 2500 — inklusiv begge grænser.

IS NULL / IS NOT NULL: WHERE mgr IS NOT NULL — brug altid IS, ikke = ved NULL.

Logiske operatorer: AND, OR, NOT — AND evalueres før OR, brug parenteser ved tvivl.

JOIN vs. subquery — hvornår bruges hvad?

Begge kan bruges til at hente data fra flere tabeller, men de har forskellig styrke:

JOIN er bedst når du vil vise kolonner fra begge tabeller i resultatet. Det er typisk mere effektivt og læsbart for simple kombinationer.

Subquery er bedst når du bruger resultatet som en betingelse eller som en enkelt værdi. F.eks. "find den medarbejder der tjener mest i sales" — her er nested subquery den naturlige løsning.

Subqueries læses indefra-ud: den inderste forespørgsel køres først og leverer en tabel/værdi til den ydre.

Mængdeoperationer — union-kompatibilitet

For at bruge UNION, INTERSECT eller EXCEPT skal de to forespørgsler returnere tabeller med samme antal kolonner, hvor parvist matchende kolonner er fra samme datatype (domæne).

UNION vs UNION ALL: UNION fjerner dubletter automatisk. UNION ALL beholder alle rækker inkl. dubletter — hurtigere men giver mere data.

EXCEPT er asymmetrisk: A EXCEPT B ≠ B EXCEPT A — rækkefølgen af forespørgslerne er vigtig.

INTERSECT og NULL: I INTERSECT behandles NULL == NULL, dvs. to NULL-værdier betragtes som ens.

Streng-funktioner og nyttige operatorer

upper(s) — konverterer til store bogstaver

lower(s) — konverterer til små bogstaver

length(s) — antal tegn i strengen

substr(s, n, m) — delstreng på m tegn fra position n. Eks: substr('DATABASE SYSTEMS', 10, 7) → 'SYSTEMS'

concat(s1, s2) — sammensætter to strenge

Aritmetik: sal * 1.55, max(SAL) - min(SAL) — kan bruges direkte i SELECT.
